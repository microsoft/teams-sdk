# Running In Teams

Now that you completed [the quickstart](../quickstart) and your agent is running locally, let's install it in Microsoft Teams. The fastest path is the [Teams CLI](/cli/).

## Prerequisites

- The CLI installed: `npm install -g @microsoft/teams.cli@preview`
- An M365 account with **custom app upload (sideloading) enabled** on the tenant
- A public HTTPS tunnel pointing at your local server (e.g. [DevTunnels](https://learn.microsoft.com/en-us/azure/developer/dev-tunnels/overview), [ngrok](https://ngrok.com/))

If you haven't run through this before, the [Quickstart: Register your app](/get-started/quickstart-register) walks the full flow end-to-end. The summary below is for developers already familiar with the steps.

## 1. Log in

```sh
teams login
teams status
```

`teams status` should report `Sideloading: enabled`. If not, your tenant admin needs to enable [custom app upload](https://learn.microsoft.com/en-us/microsoftteams/teams-custom-app-policies-and-settings).

## 2. Register the bot infrastructure

From your project directory:

```sh
teams app create \
  --name <your-bot-name> \
  --endpoint https://<tunnel-host>/api/messages \
  --env .env \
  --teams-managed
```

This creates the Entra app, a Teams-managed bot registration, and the Teams app entry in one shot. Credentials (`CLIENT_ID`, `CLIENT_SECRET`, `TENANT_ID`) land in `.env`. For C# projects use `--env appsettings.json`.

`--teams-managed` requires no Azure subscription and is ideal for prototyping. Use `--azure --resource-group <rg>` instead if you need OAuth/SSO or are heading to production. See [Bot Locations](/cli/concepts/bot-locations) for the trade-offs.

## 3. Run your agent

<LanguageInclude section="run-command" />

The bot is listening when `curl -X POST http://localhost:3978/api/messages` returns `401`.

## 4. Install the app in Teams

```sh
teams app get --install-link
```

Open the printed URL in a browser signed in to Teams, click **Add**, then send the bot a message.

## Troubleshooting

- Run `teams app doctor` against your app to surface configuration issues.
- For authentication problems, see [Authentication Troubleshooting](/teams/app-authentication/troubleshooting).
- For manual Azure setup, see [Azure Configuration](/teams/azure-configuration).

## Using Microsoft 365 Agents Toolkit instead

If you prefer working from the [Microsoft 365 Agents Toolkit](https://learn.microsoft.com/en-us/microsoftteams/platform/toolkit/install-teams-toolkit) VS Code extension, that path still works — install the extension, sign in, select **Local** under Environment Settings, and click Debug. Agents Toolkit handles tunnels, manifest stamping, and sideloading in-IDE.
